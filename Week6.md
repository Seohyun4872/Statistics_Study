### 11.1. 결측값 처리

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
**결측값의 종류**  
* 완전 무작위 결측(MCAR): 순수하게 결측값이 무작위로 발생하는 경우
* 무작위 결측(MAR): 다른 변수의 특성에 의해 해당 변수의 결측치가 체계적으로 발생한 경우
* 비무작위 결측(NMAR): 결측값들이 해당 변수 자체의 특성을 갖고 있는 경우

**해결방법**
* 표본제거방법: 전체 데이터에서 결측값 비율이 10% 미만일 땐 효율적
* 평균 대치법: 결측값을 제외한 온전한 값들의 평균을 구한 뒤 평균 값을 결측값들에 대치
* 최반값, 중앙값, 최대값, 최솟값 대치
단점: 평균을 사용하기에 통계량의 표준오차가 왜곡되어 축소되어 나타나 -> p-value가 부정확
그렇기에 위의 모든 방법들은 완전 무작위 결측이 아니라면 적절하지 않은 방식들
* 보간법: 데이터가 시계열적 특성을 가지고 있을 때 효과적, 전이나 다음시점의 값이나 평균값 등으로 대치
* 다만, 시점 인덱스 간격이 불규칙하거나 결측값이 2번 이상 연달아 있을 땐 선형적인 수치값을 계산하는 보간법 사용 필요
* 회귀 대치법: 회귀식을 이용해서 결측값을 대치 (추정하고자 하는 결측치를 가진 변수가 종속변수, 나머지 변수를 독립변수로 설정해 결측값 대치)
단점: 변수의 분산을 과소 추정하는 문제 발생
* 확률적 회귀대치법: 인위적으로 회귀식에 확률 오차항을 추가해 변동성을 조정, 관측된 값들을 변동성만큼 결측값에도 같은 변동성을 추가해주는 기법
* 다중 대치법: 지금까지 살펴본 단순 대치법들의 표본오차 과소 추정 문제를 해결하기 위해 사용되는 방법, 단순 대치를 여러번 수행해 n개의 가상적 데이터를 생성하여 이들의 평균으로 결측값을 대치
[다중 대치법 프로세스]
대치 단계: 가능한 대치 값의 분포에서 추출된 서로 다른 값으로 결측치를 처리한 n개의 데이터셋 생성 - 몬테카를로나 연쇄방정식 사용해 대치값 임의 생  
분석 단계: 생성된 각각의 데이터셋을 분석하여 모수의 추정치와 표준오차 계산  
결합 단계: 계산된 각 데이터셋의 추정치와 표준오차를 결합해 최종 결측 대치값 산출

### 11.2. 이상치 처리

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
이상치: 일부 관측치의 값이 전체 데이터의 범위에서 크게 벗어난 아주 작거나 큰 극단적인 값을 갖는 것 
전체 데이터 양이 많을 수록, 영향력 감소  
**처리 방법**
* 해당 이상치를 제거: 추정치의 분산은 감소, 실제값을 과장해 편향 발생되는 문제
* 관측값 변경
* 가중치 조정
* 도메인별로 이상치 탐지 방식이 달라짐


### 11.3. 변수 구간화

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
변수 구간화(Binning): 데이터 분석 성능 향상 또는 해석의 편리성을 위해 이산형 변수를 범주형 변수로 변환
* 동일폭, 동일 빈도 등으로 구간화
* 의미기준 구간화
* 이산 값을 평활화해 단순히 이산값으로 변화시키는 경우: 변수의 값을 일정한 폭이나 빈도로 구간을 나눈 후 각 구간안에 속한 데이터 값을 평균, 중앙값, 경곗값 등으로 변환시켜주는 것

변수 값이 효과적으로 구간화 되었는지 측정하는 법
* WOE(Weight of Evidence)
* IV(information gain): 이게 높을 수록 T/F를 잘 분류할 수 있는 정보량이 많다는 것, 즉 구간화가 잘되면 종속변수를 제대로 설명할 수 있는 IV값이 그만큼 높아지는 것임  
>> 두 수치 모두 종속변수 대비 독립변수가 예측력이 얼마나 강한지 보여주는 값들


### 11.4. 데이터 표준화와 정규화 스케일링

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
독립변수들이 서로 단위가 다르거나 편차가 심할 때 스케일을 일정한 수준으로 변환시켜주는 것
**표준화 스케일링**: 각 관측치의 값이 전체 평균을 기준으로 어느정도 떨어져있는지 나타낼 떼
<img width="233" height="123" alt="image" src="https://github.com/user-attachments/assets/b62bab8f-5c5f-4fd4-aabd-456137d280ed" />

**정규화 스케일링** : 데이터의 범위를 0-1까지로 변환해 데이터 분포를 조정하는 방식
<img width="276" height="121" alt="image" src="https://github.com/user-attachments/assets/c79a4943-e38c-458b-b158-d17e334860ac" />

특정 머신러닝 모델의 학습 효율을 증가시키기 때문에도 사용  
k-nearest Neighbor, SVM 등 거리를 활용한 군집 분석에서도 필수적  

RobustScaler: 데이터의 중앙값을 0으로 잡고 IQR 차이를 1이 되도록하는 스케일링 기법(일반적으로 표준화, 정규화보다 성능 우수)
